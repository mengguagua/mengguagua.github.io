迭代

迭代 iteration」是一种重复执行某个任务的控制结构。在迭代中，程序会在满足一定的条件下重复执行某段代码，直到这个条件不再满足。

例如累加 1+2+3+...+n

xxxxxxxxxx
/* for 循环 */
function forLoop(n) {
    let res = 0;
    // 循环求和 1, 2, ..., n-1, n
    for (let i = 1; i <= n; i++) {
        res += i;
    }
    return res;
}
/* while 循环 */
function whileLoop(n) {
    let res = 0;
    let i = 1; // 初始化条件变量
    // 循环求和 1, 2, ..., n-1, n
    while (i <= n) {
        res += i;
        i++; // 更新条件变量
    }
    return res;
}

 

递归

「递归 recursion」是一种算法策略，通过函数调用自身来解决问题。它主要包含两个阶段。

  1. 递：程序不断深入地调用自身，通常传入更小或更简化的参数，直到达到“终止条件”。

  2. 归：触发“终止条件”后，程序从最深层的递归函数开始逐层返回，汇聚每一层的结果。

例如累加 1+2+3+...+n

xxxxxxxxxx
/* 递归 */
function recur(n) {
    // 终止条件
    if (n === 1) return 1;
    // 递：递归调用
    const res = recur(n - 1);
    // 归：返回结果
    return n + res;
}

递归函数每次调用自身时，系统都会为新开启的函数分配内存，以存储局部变量、调用地址和其他信息等。这将导致两方面的结果。

  • 函数的上下文数据都存储在称为“栈帧空间”的内存区域中，直至函数返回后才会被释放。因此，递归通常比迭代更加耗费内存空间。

  • 递归调用函数会产生额外的开销。因此递归通常比循环的时间效率更低。

尾递归

如果函数在返回前的最后一步才进行递归调用，则该函数可以被编译器或解释器优化，使其在空间效率上与迭代相当。这种情况被称为「尾递归 tail recursion」

例如累加 1+2+3+...+n

xxxxxxxxxx
/* 尾递归 */
function tailRecur(n, res) {
    // 终止条件
    if (n === 0) return res;
    // 尾递归调用
    return tailRecur(n - 1, res + n);
}

许多编译器或解释器并不支持尾递归优化，例如Python

递归树

给定一个斐波那契数列 0,1,1,2,3,5,8,13,… ，求该数列的第 n 个数字。

设斐波那契数列的第 n 个数字为 f(n) ，易得两个结论。

  • 数列的前两个数字为 f(1)=0 和 f(2)=1

  • 数列中的每个数字是前两个数字的和，即 f(n)=f(n−1)+f(n−2)

递归实现如下

xxxxxxxxxx
/* 斐波那契数列：递归 */
function fib(n) {
    // 终止条件 f(1) = 0, f(2) = 1
    if (n === 1 || n === 2) return n - 1;
    // 递归调用 f(n) = f(n-1) + f(n-2)
    const res = fib(n - 1) + fib(n - 2);
    // 返回结果 f(n)
    return res;
}

递归体现“将问题分解为更小子问题”的思维模式。

  • 从算法角度看，搜索、排序、回溯、分治、动态规划等许多重要算法策略都直接或间接地应用这种思维方式。

  • 从数据结构角度看，递归天然适合处理链表、树和图的相关问题。

栈和递归的相似性

迭代和递归具有的内在联系，以上述的递归函数为例，求和操作在递归的“归”阶段进行。这意味着最初被调用的函数实际上是最后完成其求和操作的，这种工作机制与栈的“先入后出”原则是异曲同工的。

  1. 递：当函数被调用时，系统会在“调用栈”上为该函数分配新的栈帧，用于存储函数的局部变量、参数、返回地址等数据。

  2. 归：当函数完成执行并返回时，对应的栈帧会从“调用栈”上被移除，恢复之前函数的执行环境。

使用一个显式的栈来模拟调用栈的行为，从而将递归转化为迭代形式:

xxxxxxxxxx
/* 使用迭代模拟递归 */
int forLoopRecur(int n) {
    // 使用一个显式的栈来模拟系统调用栈
    Stack<Integer> stack = new Stack<>();
    int res = 0;
    // 递：递归调用
    for (int i = n; i > 0; i--) {
        // 通过“入栈操作”模拟“递”
        stack.push(i);
    }
    // 归：返回结果
    while (!stack.isEmpty()) {
        // 通过“出栈操作”模拟“归”
        res += stack.pop();
    }
    // res = 1+2+3+...+n
    return res;
}

 

时间复杂度

时间复杂度分析统计的不是算法运行时间，而是算法运行时间随着数据量变大时的增长趋势。(统计时间增长趋势)

xxxxxxxxxx
// 算法 a 的时间复杂度：线性阶
function algorithm_B(n) {
    for (let i = 0; i < n; i++) {
        console.log(0);
    }
}
// 算法 b 的时间复杂度：常数阶
function algorithm_C(n) {
    for (let i = 0; i < 1000000; i++) {
        console.log(0);
    }
}

  • 算法 a 中的打印操作需要循环 n 次，算法运行时间随着 n 增大呈线性增长。此算法的时间复杂度被称为“线性阶”。

  • 算法 b 中的打印操作需要循环 1000000 次，虽然运行时间很长，但它与输入数据大小 b 无关，为“常数阶”。

线性阶的时间复杂度记为 O(n) ，这个数学符号称为「大 O 记号 big-O notation」，表示函数 T(n) 的「渐近上界 asymptotic upper bound」。同样的渐进下界就是Ω，平均复杂度是Θ 。

O的读音是：欧米可荣。补充：Ω读 欧米伽，Θ读 西塔

时间复杂度也存在一定的局限性。例如，尽管算法 a 的时间复杂度比 b 高，但在输入数据大小 n 较小时，算法 a 明显优于算法 b 。在这些情况下，很难仅凭时间复杂度判断算法效率的高低。

怎么计算时间复杂度

第一步：统计操作数量

  1. 忽略 T(n) 中的常数项。因为它们都与 n 无关，所以对时间复杂度不产生影响。

  2. 省略所有系数。例如，循环 2n 次、5n+1 次等，都可以简化记为 n 次，因为 n 前面的系数对时间复杂度没有影响。

  3. 循环嵌套时使用乘法。总操作数量等于外层循环和内层循环操作数量之积，每一层循环依然可以分别套用第 1. 点和第 2. 点的技巧。

例如：

xxxxxxxxxx
function algorithm(n) {
    let a = 1;  // +0（忽略常数）
    a = a + n;  // +0（忽略常数）
    // +n（忽略系数）
    for (let i = 0; i < 5 * n + 1; i++) {
        console.log(0);
    }
    // +n*n（循环嵌套算乘法）
    for (let i = 0; i < 2 * n; i++) {
        for (let j = 0; j < n + 1; j++) {
            console.log(0);
        }
    }
}

所以可以算出是：T(n) = n² + n

第二步：判断渐近上界

时间复杂度由多项式 T(n) 中最高阶的项来决定。这是因为在 n 趋于无穷大时，最高阶的项将发挥主导作用，其他项的影响都可以被忽略。例如图：

常见的时间复杂度类型

常见的时间复杂度，在实际场景中用到的为：

常数阶：普通函数

线性阶：迭代函数，n关联迭代次数，如遍历数组，链表

平方阶：循环嵌套的函数，如冒泡排序

指数阶：递归函数，如递归树(每次一分为二)

对数阶：递归函数，每次缩减一半

线性对数阶：迭代循环和每次缩减一半的情况，如快速排序、归并排序、堆排序

 

空间复杂度

「空间复杂度 space complexity」用于衡量算法占用内存空间随着数据量变大时的增长趋势。这个概念与时间复杂度非常类似，只需将“运行时间”替换为“占用内存空间”。

算法在运行过程中使用的内存空间主要包括以下几种。

  • 输入空间：用于存储算法的输入数据。

  • 暂存空间：用于存储算法在运行过程中的变量、对象、函数上下文等数据。

    暂存空间可以进一步划分为三个部分。

    暂存数据：用于保存算法运行过程中的各种常量、变量、对象等。

    栈帧空间：用于保存调用函数的上下文数据。系统在每次调用函数时都会在栈顶部创建一个栈帧，函数返回后，栈帧空间会被释放。

    指令空间：用于保存编译后的程序指令，在实际统计中通常忽略不计。

  • 输出空间：用于存储算法的输出数据。

所以一般情况，计算空间复杂度就是通常统计暂存数据、栈帧空间和输出数据三部分

如下：

xxxxxxxxxx
/* 类 */
class Node {
    val;
    next;
    constructor(val) {
        this.val = val === undefined ? 0 : val; // 节点值
        this.next = null;                       // 指向下一节点的引用
    }
}
/* 函数 */
function constFunc() {
    // 执行某些操作
    return 0;
}

    const a = 0;              // 暂存数据（常量）

    const node = new Node(0); // 暂存数据（对象）

}
怎么计算空间复杂度

  1. 以最差输入数据为准：假设函数当 n<10 时，空间复杂度为 n(1) ；但当 n>10 时，占用 O(n) 空间；取最差空间复杂度为 O(n) 。

  2. 以算法运行中的峰值内存为准：例如，程序在执行最后一行之前，占用 n(1) 空间；当初始化数组时，程序占用 O(n) 空间；取最差空间复杂度为 O(n) 。

例如：

xxxxxxxxxx
// 复杂度为 O(n)
function algorithm(n) {
    const a = 0;                   // O(1)
    const b = new Array(10000);    // O(1)
    if (n > 10) {
        const nums = new Array(n); // O(n)
    }
}
// 注意栈帧空间
function constFunc() {
    // 执行某些操作
    return 0;
}
/* 每次循环都返回并释放了栈帧空间，所以是 O(1) */
function loop(n) {
    for (let i = 0; i < n; i++) {
        constFunc();
    }
}
/* 递归每次循环都存在recur()，所以是 O(n) */
function recur(n) {
    if (n === 1) return;
    return recur(n - 1);
}
常见的空间复杂度类型

常数阶，空间复杂与输入无关。

需要注意的是，在循环中初始化变量或调用函数而占用的内存，在进入下一循环后就会被释放，因此不会累积占用空间，空间复杂度仍为 n(1) 

xxxxxxxxxx
/* 函数 */
function constFunc() {
    // 执行某些操作
    return 0;
}
/* 常数阶 */
function constant(n) {
    // 常量、变量、对象占用 O(1) 空间
    const a = 0;
    const b = 0;
    const nums = new Array(10000);
    const node = new ListNode(0);
    // 循环中的变量占用 O(1) 空间
    for (let i = 0; i < n; i++) {
        const c = 0;
    }
    // 循环中的函数占用 O(1) 空间
    for (let i = 0; i < n; i++) {
        constFunc();
    }
}

线性阶，常见于元素数量与 n 成正比的数组、链表、栈、队列等，如：

数据结构

数据结构的逻辑结构，分为线性与非线性，常见的如下图：

  • 线性数据结构：数组、链表、栈、队列、哈希表。

  • 非线性数据结构：树、堆、图、哈希表。

数据结构的物理结构，分为连续和分散

内存和硬盘是两种主要的存储硬件设备。硬盘主要用于长期存储数据，容量较大（通常可达到 TB 级别）、速度较慢。内存用于运行程序时暂存数据，速度快，但容量较小（通常为 GB 级别）。

在算法运行过程中，相关数据都存储在内存中，如图展示了一个计算机内存条，其中每个黑色方块都包含一块内存空间，内存空间可以想象成单元格表，在算法运行时，所有数据都被存储在这些单元格中。

系统通过内存地址来访问目标位置的数据，计算机根据特定规则为表格中的每个单元格分配编号，确保每个内存空间都有唯一的内存地址，通过内存地址找到数据。

内存是所有程序的共享资源，当某块内存被某个程序占用时，则无法被其他程序同时使用。因此在数据结构与算法的设计中，内存资源是一个重要的考虑因素

物理结构反映了数据在计算机内存中的存储方式，可分为连续空间存储（数组）和分散空间存储（链表）。物理结构从底层决定了数据的访问、更新、增删等操作方法

所有数据结构都是基于数组、链表或二者的组合实现的

  • 基于数组可实现：栈、队列、哈希表、树、堆、图、矩阵、张量（维度 ≥3 的数组）等。

  • 基于链表可实现：栈、队列、哈希表、树、堆、图等

基本数据类型

基本数据类型是 CPU 可以直接进行运算的类型，在算法中直接被使用，主要包括以下几种类型。

  • 整数类型 byte、short、int、long 。

  • 浮点数类型 float、double ，用于表示小数。

  • 字符类型 char ，用于表示各种语言的字母、标点符号、甚至表情符号等。

  • 布尔类型 bool ，用于表示“是”与“否”判断。

基本数据类型以二进制的形式存储在计算机中。一个二进制位即为 1 比特。在绝大多数现代系统中，1 字节（byte）由 8 比特（bits）组成。

基本数据类型的取值范围取决于其占用的空间大小。下面以 Java 为例。

  • 整数类型 byte 占用 1 byte = 8 bits ，可以表示 28 个数字。

  • 整数类型 int 占用 4 bytes = 32 bits ，可以表示 232 个数字。

Q: 1byte有8位，为什么是2⁷，而不是2的8次方？

A: 应为计算机将数字的二进制表示的最高位视为符号位， 0 表示正数，1 表示负数，其余位表示数字的值。

Q: 基本数据类型与数据结构之间有什么联系？

A: 如果想要表示“一排数字”，自然会想到使用数组。这是因为数组的线性结构可以表示数字的相邻关系和顺序关系，但至于存储的内容是整数 int、小数 float 或是字符 char ，则与“数据结构”无关。

换句话说，基本数据类型提供了数据的“内容类型”，而数据结构提供了数据的“组织方式”。

数字编码(推荐理解)

首先我们指出，数字是以“补码”的形式存储在计算机中的(整数)。

下面解释，为什么这么做，首先记住三个概念。这是一种基础设定。

  • 原码：将数字的二进制表示的最高位视为符号位，其中 0 表示正数，1 表示负数，其余位表示数字的值。

  • 反码：正数的反码与其原码相同，负数的反码是对其原码除符号位外的所有位取反。

  • 补码：正数的补码与其原码相同，负数的补码是在其反码的基础上加 1 。

「原码」虽然最直观，但存在一些局限性。一方面，负数的原码不能直接用于运算。

例如在原码下计算 1+(−2) ，得到的结果是 −3 ，这显然是不对的。

xxxxxxxxxx
1+(−2) 
=> 0000 0001 + 1000 0010
= 1000 0011
=> -3

为了解决此问题，计算机设计者引入「反码」概念。如果先将原码转换为反码，并在反码下计算 1+(−2) ，最后将结果从反码转化回原码，则可得到正确结果 −1 。即数字 => 原码 =>反码相加 => 原码

xxxxxxxxxx
1+(−2) 
=> 0000 0001 + 1000 0010 // 原码
= 0000 0001 + 1111 1101 // 反码
= 1111 1110 // 反码
= 1000 0001 // 原码
=> -1

另一方面，数字零的原码有 +0 和 −0 两种表示方式。这意味着数字零对应着两个不同的二进制编码，其可能会带来歧义。比如在条件判断中，如果没有区分正零和负零，则可能会导致判断结果出错。而如果我们想要处理正零和负零歧义，则需要引入额外的判断操作，其会降低计算机的运算效率。

xxxxxxxxxx
+0 => 0000 0000
-0 => 1000 0000

同理，反码也有正负零的歧义，因此计算机设计者又巧妙的引入了补码概念。

xxxxxxxxxx
-0 
=> 1000 0000 // 原码
=> 1111 1111 // 反码
=> 1 0000 0000 // 补码
= 0000 0000 // 1byte只有8位，最高位舍弃
= 0

在负零的反码基础上加 1 会产生进位，但 byte 类型的长度只有 8 位，因此溢出到第 9 位的 1 会被舍弃。也就是说，负零的补码为 00000000 ，与正零的补码相同。

Q：为什么1byte 区间是 -128到127，而不是-127到127？

A：因为存在一个特殊的补码，1000 0000。按规则倒推，这个补码的原码应该是0111 111 => 0000 0000 => 0。这显然是矛盾的，所以计算机设计者规定，这个特殊的补码就是表示-128。实际上也运算的通。

xxxxxxxxxx
-128
=> (-127) + (-1)
=> 1111 1111 + 1000 0001 //原码
=> 1000 0000 + 1111 1110 // 反码
=> 1000 0001 + 1111 1111 // 补码
= 1 1000 0000 // 补码
= 1000 0000 // 进位舍最高位

Q：为什么都是二进制加法运算？

A：计算机内部的硬件电路主要是基于加法运算设计的。这是因为加法运算相对于其他运算（比如乘法、除法和减法）来说，硬件实现起来更简单，更容易进行并行化处理，运算速度更快。

通过将加法与一些基本逻辑运算结合，计算机能够实现各种其他的数学运算。例如，计算减法 a−b 可以转换为计算加法 a+(−b) ；计算乘法和除法可以转换为计算多次加法或减法。

 

数组/链表

数组

数组存储在连续的内存空间内。这种做法，系统可以利用来优化数据结构的操作效率。

  • 空间效率高: 数组为数据分配了连续的内存块，无须额外的结构开销。

  • 支持随机访问: 数组允许在 O(1) 时间复杂度内访问任何元素。

  • 缓存局部性: 当访问数组元素时，计算机不仅会加载它，还会缓存其周围的其他数据，从而借助高速缓存来提升后续操作的执行速度。

连续空间存储，其存在以下缺点。

  • 插入与删除效率低:当数组中元素较多时，插入与删除操作需要移动大量的元素。平均时间复杂度均为 O(n)。

Tip: 其它语言如java，数组不可变，就有一下缺点，但js中默认数组可变，所以实用性很高。

长度不可变: 数组在初始化后长度就固定了，扩容数组需要将所有数据复制到新数组，开销很大。

空间浪费: 如果数组分配的大小超过了实际所需，那么多余的空间就被浪费了。

链表

内存空间是所有程序的公共资源，空闲的内存空间可能散落在内存各处。而存储数组的内存空间必须是连续的，而当数组非常大时，内存可能无法提供如此大的连续空间。此时需要，链表来灵活性存储。

「链表 linked list」是一种线性数据结构，链表的内存地址是无须连续的。其中的每个元素都是一个节点对象，各个节点通过“引用”相连接。

  • 存储灵活：链表的内存地址是无须连续的

  • 插入与删除效率高：只要改变引用就可以改变顺序，平均时间复杂度均为 O(1)。

不连续空间存储，其存在以下缺点。

  • 链表节点除了包含值，还需额外保存一个引用（指针）。因此在相同数据量下，链表比数组占用更多的内存空间。

  • 查询访问效率低：需要从头节点出发，逐个向后遍历，直至找到目标节点，时间复杂度均为 O(n)。

 

小结：在js语言环境下，数组和链表都是很高效的数据结构，因为常数和线性的时间复杂度，都是可以接受的。

 

栈/队列

栈

「栈 stack」是一种遵循先入后出的逻辑的线性数据结构，栈可以被视为一种受限制的数组或链表。

应用例子：

浏览器中的后退与前进、软件中的撤销与反撤销。每当我们打开新的网页，浏览器就会将上一个网页执行入栈，这样我们就可以通过后退操作回到上一页面。后退操作实际上是在执行出栈。如果要同时支持后退和前进，那么需要两个栈来配合实现。

队列

「队列 queue」是一种遵循先入先出规则的线性数据结构，队列也可以被视为一种受限制的数组或链表。

应用例子：

各类待办事项。任何需要实现“先来后到”功能的场景，例如打印机的任务队列、餐厅的出餐队列等。队列在这些场景中可以有效地维护处理顺序

 

哈希表

「哈希表 hash table」，又称「散列表」，其通过建立键 key 与值 value 之间的映射，实现高效的元素查询。具体而言，我们向哈希表输入一个键 key ，则可以在 O(1) 时间内获取对应的值 value 。

映射到js环境，就是对象{}，map之类，数据结构可以理解为[{key1: v1}, {key2:v2}, ..., {keyn:vn}]，数组每个对象取名叫「bucket」，会存放一对键值，查询操作就是输入key，通过哈希函数输出bucket的下标索引，通过索引获取bucket。过程类似上图。

哈希冲突，理论上bucket的下标索引是有限的，例如100个对象。但可输入的key远远大于这个范围，所以会存在，输入不同的key，经过哈希函数后得到相同的下标索引的问题。

哈希函数定义：哈希函数是接受输入（通常是字符串）并生成数字的函数。如果使用相同的输入多次调用哈希函数，它将始终返回相同的数字，并且返回的数字始终在函数算法承诺的范围内。

所以解决哈希冲突推荐的做法是，优化哈希函数算法，例如使用murmur3(这类问题，一般不用考虑，设计语言本身会解决)

哈希函数参考文章：https://samwho.dev/hashing/

如上图是通过修改数据结构的形式，解决哈希冲突。一个bucket不再是一对键值，而是一组链表。不同的哈希函数输入，得到相同的bucket下标索引后(冲突情况)，将键值对依次往链表里插入。

查询逻辑就演化为，输入key，通过哈希函数得到bucket下标索引，通过下标索引得到一组链表，然后依次遍历链表，对比key，获取value。显然多了遍历链表的过程，查询的时间复杂度就不再是O(1)，而是O(n)。解决了冲突，牺牲了部分的效率。

 

堆

从Top-K 问题出发，理解什么是堆。

Top-K 问题：给定一个长度为 n 无序数组 nums ，请返回数组中前 k 大的元素。例如几百万条微博消息中找到点击量最大的10条。

方法一：遍历选择

每次顺序遍历n个元素，比较得到最大的一个，k个就遍历k次。时间复杂度为O(nk)，k很大时候趋近于O(n²)，消耗比较大。

方法二：排序

将数组从小到大排序，取最右边前k个元素，这么做额外对k个以外的元素进行了顺序化，对应的时间复杂度固定是O(n log n)

方法三：堆

「堆 heap」是一种满足特定条件的完全二叉树。

完全二叉树的定义是：只有最底层的节点未被填满，且最底层节点尽量靠左填充的二叉树。

所以堆的定义是：

  • 最底层节点靠左填充，其他层的节点都被填满。

  • 将二叉树的根节点称为“堆顶”，将底层最靠右的节点称为“堆底”。

  • 任意节点的值都 ≥ 或 ≤ 其子节点的值。

    任意节点的值 ≥ 其子节点的值，叫「大顶堆 max heap」

    任意节点的值 ≤ 其子节点的值，叫「小顶堆 min heap」

所以堆有以下特性：

逻辑理解上，堆的结构如上图左侧，实际实现上还是数组存储，只是数组下标索引不是依次递增，而是有以下规则：

已知当前索引 i，其左子节点索引就是 2i+1 ，右子节点索引为 2i+2 ，父节点索引为 (i−1)/2（向下取整）。当索引越界时，表示空节点或节点不存在。

应用：如解决Top-K问题，已知堆的概念，现在我们有[7,1,6,3,2]数组，取前3个最大的数，利用堆的做法是

  1. 初始化一个空的堆(小顶堆)，其堆顶元素最小。这时没有时间复杂度开销

  2. 将数组的前 k 个元素建堆，建堆的时间复杂度是O(n)，这里就是O(k)

  3. 从第 k+1 个元素开始，若当前元素大于堆顶元素，则将堆顶元素出堆，并将当前元素入堆。出堆，入堆都要重新排堆结构，时间复杂度都是log(n)，一轮出入是2log(n)，按时间复杂度计算，忽略常数，一次出入堆就是log(k)。

  4. 遍历完成后，堆中保存的就是最大的 k 个元素。这里遍历次数就是 (n-k)次，复杂度就是O( (n-k)log(k) )

计算上面总的时间复杂度就是 O(k) + O( (n-k) log(k) ) , 参考之前内容，时间复杂度计算规则，当k趋近于1，k越小，时间复杂度可为O(n)；k趋近于n/2，时间复杂度为O( nlog(n) )。

总结：时间复杂度为O( nlog(n) ) 到 O(n)

注：建堆复杂度是O(n)，出堆和入堆(重排)复杂度是O(log(n))

搜索算法

二分查找

「二分查找 binary search」是一种基于分治策略的高效搜索算法。它利用数据的有序性，每轮减少一半搜索范围，直至找到目标元素或搜索区间为空为止。

二分查找的逻辑是：从一个有序排列的数组nums里查找到目标值(target)的索引的逻辑。

现有n个数据，取第一个数据下标 i=0, 取最后一个数据下标 i=n-1，取中间数据下标 m=(i+j)/2。比较目标值(target)和中间值nums[m]的大小。

  1. 当 nums[m] < target 时，说明 target 在区间 [m+1, j] 中，因此执行 i=m+1 。

  2. 当 nums[m] > target 时，说明 target 在区间 [i, m−1] 中，因此执行 j=m−1 。

  3. 当 nums[m] = target 时，说明找到 target ，因此返回索引 m。

复杂度：时间复杂度为O(log n)，效率很高。

缺点：1、需要有序的数据，不是有序的为了二分法而先去排序，开销就很大。2、需要数组结构，需要频繁查询和比较数据，如果是链表结构，每次访问的开销是O(n)，不合适用二分法。3、n很小时候不适用，二分法需要加法，除法和判断，一次处理开销大。

哈希优化查找

将线性查找替换为哈希查找来降低算法的时间复杂度

如有一个问题：给定一个整数数组 nums 和一个目标元素 target ，请在数组中搜索和为 target 的两个元素，并返回它们的数组索引。

1、一般逻辑思维是写两层循环，判断拿到的两个数的和是不是等于target，是则返回俩者的下标。这种算法的时间复杂度是O(n²)，n偏大时不推荐。

xxxxxxxxxx
/* 方法一：枚举 */
function twoSumBruteForce(nums, target) {
    const n = nums.length;
    // 两层循环，时间复杂度 O(n^2)
    for (let i = 0; i < n; i++) {
        for (let j = i + 1; j < n; j++) {
            if (nums[i] + nums[j] === target) {
                return [i, j];
            }
        }
    }
    return [];
}

2、另一个思路是额外多建立一个对象(哈希表)，遍历一层，将每次遍历的值进行如下计算和判断

  • 判断数字 target - nums[i] 是否在哈希表中，若是则直接返回这两个元素的索引。

  • 将键值对 nums[i] 和索引 i 添加进哈希表(注意：key是元素值，value是索引)

xxxxxxxxxx
/* 方法二：辅助哈希表 */
function twoSumHashTable(nums, target) {
    // 辅助哈希表，空间复杂度 O(n)
    let m = {};
    // 单层循环，时间复杂度 O(n)
    for (let i = 0; i < nums.length; i++) {
        if (m[target - nums[i]] !== undefined) {
            return [m[target - nums[i]], i];
        } else {
            m[nums[i]] = i;
        }
    }
    return [];
}

这种方式，时间复杂度是O(n)，因为m可能设置n个值，空间复杂度也是O(n)。整体效率都不错。

 

排序

选择排序

「选择排序 selection sort」的工作原理：开启一个循环，每轮从未排序区间选择最小的元素，将其放到已排序区间的末尾。

设数组的长度为 n

  1. 初始状态下，所有元素未排序，即未排序（索引）区间为 [0,n−1] 。

  2. 选取区间 [0,n−1] 中的最小元素，将其与索引 0 处元素交换。完成后，数组前 1 个元素已排序。

  3. 选取区间 [1,n−1] 中的最小元素，将其与索引 1 处元素交换。完成后，数组前 2 个元素已排序。

  4. 以此类推。经过 n−1 轮选择与交换后，数组前 n−1 个元素已排序。

  5. 仅剩的一个元素必定是最大元素，无须排序，因此数组排序完成。

时间复杂度为 O(n²)

冒泡排序

「冒泡排序 bubble sort」通过连续地比较与交换相邻元素实现排序。这个过程就像气泡从底部升到顶部一样，因此得名冒泡排序。

冒泡过程可以利用元素交换操作来模拟：从数组最左端开始向右遍历，依次比较相邻元素大小，如果“左元素 > 右元素”就交换它俩。遍历完成后，最大的元素会被移动到数组的最右端，如此反复。

设数组的长度为 n

  1. 首先，对 n 个元素执行“冒泡”，将数组的最大元素交换至正确位置，

  2. 接下来，对剩余 n−1 个元素执行“冒泡”，将第二大元素交换至正确位置。

  3. 以此类推，经过 n−1 轮“冒泡”后，前 n−1 大的元素都被交换至正确位置。

  4. 仅剩的一个元素必定是最小元素，无须排序，因此数组排序完成。

特别的：增加一个标记flag判断是否有过交换操作，没有交换过，证明完成排序，不用再继续比较。

时间复杂度为 O(n²) 小概率到 O(n)

插入排序

推荐掌握

「插入排序 insertion sort」具体来说，是在未排序区间选择一个基准元素，将该元素与其左侧已排序区间的元素逐一比较大小，并将该元素插入到正确的位置。

  1. 初始状态下，数组的第 1 个元素已完成排序。

  2. 选取数组的第 2 个元素作为 base ，将其插入到正确位置后，数组的前 2 个元素已排序(如从小到大)。

  3. 选取第 3 个元素作为 base ，将其插入(从base左边的最后一个数开始比较)到正确位置后，数组的前 3 个元素已排序(如从小到大)。

  4. 以此类推，在最后一轮中，选取最后一个元素作为 base ，将其插入到正确位置后，所有元素均已排序。

xxxxxxxxxx
// 建议到浏览器f12，调试查看
function insertionSort(nums) {
    // i从1开始，将第一个数设置为已完成排序的部分
    for (let i = 1; i < nums.length; i++) {
        // 提前保留base值。j是指前j个已经完成排序的数
        let base = nums[i],
            j = i - 1;
        // 内循环：将 base 插入到已排序部分的正确位置
        while (j >= 0 && nums[j] > base) { 
          debugger;
          nums[j + 1] = nums[j]; // 将 nums[j] 向右移动一位
          j--;
        }
        nums[j + 1] = base; // 将 base 赋值到正确位置
    }
};
let numss = [3,2,4,1,9,6]; 
insertionSort(numss); 
console.log(numss)

时间复杂度为 O(n²) 到 O(n)

优点：在数据量n较小的情况下，插入排序通常更快，例如Java内置到排序方法，在短数组情况下就是采用的插入排序。

快速排序

推荐掌握

「快速排序 quick sort」是一种基于分治策略的排序算法。

快速排序的核心操作是“哨兵划分”，其目标是：选择数组中的某个元素作为“基准数”，将所有小于基准数的元素移到其左侧，而大于基准数的元素移到其右侧。

  1. 选取数组最左端元素作为基准数，初始化两个指针 i 和 j 分别指向数组的两端。

  2. 设置一个循环，在每次循环中，先用 j向左( j-- )寻找第一个比基准数小的元素，然后i向右( i++ )寻找第一个比基准数大的元素，交换这两个元素的位置。（如上图，第一次循环，j走到 0 位置，i走到 4 位置，然后交换0和4的位置）

  3. 循环执行步骤 2 ，直到 i 和 j 相遇时停止，最后将基准数交换至两个子数组的分界线。

  1. 通过1-3步骤，划分出左子树和右子树，分别再对两棵子树进行1-3的递归步骤。直到子树长度为1位置就是排序完成。

xxxxxxxxxx
/* 元素交换 */
function swap(nums, i, j) {
    let tmp = nums[i];
    nums[i] = nums[j];
    nums[j] = tmp;
}


/* 哨兵划分 */
function partition(nums, left, right) {
    // 以 nums[left] 作为基准数
    let i = left,
        j = right;
    while (i < j) {
        while (i < j && nums[j] >= nums[left]) {
            j -= 1; // 从右向左找首个小于基准数的元素
        }
        while (i < j && nums[i] <= nums[left]) {
            i += 1; // 从左向右找首个大于基准数的元素
        }
        // 元素交换
        this.swap(nums, i, j); // 交换这两个元素
    }
    this.swap(nums, i, left); // 将基准数交换至两子数组的分界线
    return i; // 返回基准数的索引
}
/* 快速排序 */
function quickSort(nums, left, right) {
    // 子数组长度为 1 时终止递归
    if (left >= right) return;
    // 哨兵划分
    const pivot = this.partition(nums, left, right);
    // 递归左子数组、右子数组
    this.quickSort(nums, left, pivot - 1);
    this.quickSort(nums, pivot + 1, right);
}
let arrs = [3,1,8,6,5,2,20]; 
quickSort(arrs,0,6); 
console.log(arrs); //  [1, 2, 3, 5, 6, 8, 20]

时间复杂度为: O(n²) 到 O(nlog n) 。只有每次划分左右子树时，每次一边长度都是0，一边长度都是n-1，复杂度会为O(n²)

优点：时间复杂度基本稳定在O(nlog n) ，最差情况出现概率很低。没有使用链表，元素访问效率高。Java内置到排序方法，在长数组情况下就是采用快速排序

优化：如果基准数在数组里属于特别小，或者特别大，复杂度容易趋近O(n²) 。所以可以优化选取基准数的方式，不默认取数组第一个为基准，而是取数组头，中，尾三个，取值中间大小的元素为基准数。

桶排序

假设有一个百万级别的数据，在一台机器上不好直接排序，则可以使用如下方法。

考虑一个长度为 n 的数组，元素是范围 [0,1) 的浮点数

  1. 初始化 k 个桶，将 n 个元素分配到 k 个桶中

  2. 对每个桶分别执行排序（采用语言的内置排序函数）

  3. 按照桶的从小到大的顺序，合并结果。

时间复杂度 O(n+k) ：假设元素在各个桶内平均分布，那么每个桶内的元素数量为 n/k 。假设排序单个桶使用 O(n/k log⁡ n/k) 时间，则排序所有桶使用O(n log⁡ n/k)时间。当桶数量 k 比较大时，时间复杂度则趋向于 O(n)

这种做法的重点是k数量要够多，且数据接近平均分布。

 

其它

其它排序方式还有，归并排序，堆排序，计数排序，基数排序等。

归并排序和快速排序有相似点，都是分治策略，特点是不断分化数组，直到长度为1，重排短数组，再合并成长数组，但总体速度不如快速排序，不做赘述。时间复杂度O(nlog n)

堆排序也是分治策略，见前文堆的Top-K问题，把k替换成数组长度n即可，不做赘述。时间复杂度O(nlog n)

计数排序和基数排序类似，前提都是非负整数，条件适用性差，不做赘述。

回溯/动态规划/贪心

这3个是算法的思考方式，前端实际业务开发中出现不多，这里简单描述，有兴趣自行谷歌。

回溯算法本质是穷举法，不断分解问题，解决最下层小问题，再基于小问题的解，不断向上回溯得到解，最终得到大问题的解(使用递归)。在穷举的同时通过问题的一些约束条件，裁剪掉部分穷举，这种做法效率比较低，但最符合人的思考习惯。

动态规划可以理解为穷举的进阶思考方式，将一个问题分解为一系列更小的子问题，但子问题重叠出现，通过避免重复计算子问题，从而大幅提升时间效率。

动态规划思考方式是，先判断这个问题是不是可以通过穷举解决(暴力搜索)，然后就去尝试穷举，然后总结规律，判断是不是有重叠子问题，将重叠的部分提取和复用(记忆化搜索)，考虑是不是还有算法优化余地，最终实现自顶而下的动态规划算法。

贪心算法，基本思想是在问题的每个决策阶段，都选择当前看起来最优的选择，即贪心地做出局部最优的决策，以期望获得全局最优解。贪心算法操作直接、实现简单，而且通常效率也很高。

贪心算法问题：例如 硬币额度=[1,20,50]：假设 目标凑齐额度=60 ，贪心算法只能找到 50+1×10 的兑换组合，共计 11 枚硬币，但动态规划可以找到最优解 20+20+20 ，仅需 3 枚硬币。
